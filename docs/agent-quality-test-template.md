# Agent 出力品質テスト・評価テンプレート

Copilot PR Reviewer（およびその他の LLM Agent）の出力品質を評価するテンプレート。

## 目的

Agent が提案したレビューコメント・改善指摘が実装に値するか、妥当性や精度を人間が検証します。

---

## 📋 評価実施フロー

```
Agent がコメントを生成
  ↓
このテンプレートで評価
  ↓
スコア を agent-performance.md に記載
  ↓
次回改善に活かす
```

---

## ✅ 評価チェックリスト

PR レビューを受けた際、以下の項目を評価してください。

### 【精度】コメントが正確か？

| 項目 | YES | NO | 備考 |
|-----|-----|-----|------|
| コメントの指摘内容は正確か？ | [ ] | [ ] | 実装に基づいた指摘か、推測か |
| 提示されたコードサンプルは実行可能か？ | [ ] | [ ] | 実装時に即座に使えるレベルか |
| セキュリティ指摘は的確か？ | [ ] | [ ] | 実際の脆弱性か、過度な懸念か |
| パフォーマンス指摘は定量的か？ | [ ] | [ ] | 「遅い」だけでなく、根拠があるか |

**小計**: YES 数 / 総数 = **精度スコア** ✓

---

### 【価値】コメント対応に価値があるか？

| 項目 | YES | NO | 備考 |
|-----|-----|-----|------|
| 指摘は実装品質を向上させるか？ | [ ] | [ ] | 本当に対応価値があるか |
| 優先度は適切に分類されているか？ | [ ] | [ ] | Critical は本当に緊急か |
| 実装コストに見合う価値があるか？ | [ ] | [ ] | 時間がかかる割に効果が薄くないか |
| スタイル統一だけの指摘は適切に分類されているか？ | [ ] | [ ] | Nice-to-have に分類されているか |

**小計**: YES 数 / 総数 = **価値スコア** ✓

---

### 【完全性】必要な指摘がもれていないか？

| 項目 | YES | NO | 備考 |
|-----|-----|-----|------|
| テスト不足の指摘はあるか？ | [ ] | [ ] | カバレッジ・エッジケースなど |
| エラーハンドリングの指摘はあるか？ | [ ] | [ ] | 異常系パスの処理について |
| ドキュメント更新の指摘はあるか？ | [ ] | [ ] | 振る舞い変更があればドキュメント更新は必須 |
| バックエンド・フロントエンド間の整合性指摘はあるか？ | [ ] | [ ] | API 契約に基づいた指摘 |

**小計**: YES 数 / 総数 = **完全性スコア** ✓

---

### 【追加評価】Agent の動作について

| 項目 | スコア | 備考 |
|-----|--------|------|
| **理解度**: 実装内容を理解しているか？ | 1-5 | コメントの文脈から判断 |
| **有用性**: 開発者の時間を削減したか？ | 1-5 | コメント対応が効率化に寄与したか |
| **説得力**: コメントの説得力は十分か？ | 1-5 | 開発者が納得して対応できたか |

---

## 🎯 総合品質スコア計算

```
品質スコア = (精度 + 価値 + 完全性) / 3

スコア表：
- 3.5 以上: ⭐⭐⭐⭐⭐ (優秀)
- 3.0 以上: ⭐⭐⭐⭐ (良好)
- 2.5 以上: ⭐⭐⭐ (標準)
- 2.0 以上: ⭐⭐ (要改善)
- 2.0 未満: ⭐ (不適切)
```

## 📝 サンプル評価記録

### PR #29 評価例（/admin/settings ページ実装）

**コメント例 #1**: "decodeAdminToken() の throw に try/catch がない"

#### 精度
- [x] 指摘が正確: YES - 実装を読むと確かに try/catch がない
- [x] コード例が実行可能: YES - 提示された try/catch コードはそのまま使える
- [x] 理由が明確: YES - "JWTが不正な場合、useEffect でクラッシュ"
- **精度スコア**: 3/3 = 100%

#### 価値
- [x] 品質向上: YES - エラーハンドリングは重要
- [x] 優先度妥当: YES - **Critical** として分類されている
- [x] 実装コスト妥当: YES - 数分の修正で済む
- [x] 分類が適切: YES - 機能問題として分類
- **価値スコア**: 4/4 = 100%

#### 完全性
- [x] テスト不足の指摘: YES - テスト追加も対応した
- [x] エラーハンドリング: YES - こまさにこのコメント
- [x] ドキュメント: - (不要)
- [x] 整合性: - (フロント単独)
- **完全性スコア**: 2/2 = 100%

#### 総合スコア
```
(100% + 100% + 100%) / 3 = 100%
→ ⭐⭐⭐⭐⭐ (優秀)
```

## 🔄 使用方法

### PR ごとの評価実施

1. PR レビューコメント受け取り直後
2. このテンプレートを使って各コメントを評価
3. スコアを計算
4. `docs/agent-performance.md` に品質スコアを記載

### 月次フィードバック

毎月末に全 PR の品質スコアを集計し、Agent のプロンプト・設定改善に活かします。

---

## 📊 結果の活用方法

### スコアが高い場合（⭐⭐⭐⭐⭐ ~ ⭐⭐⭐⭐）
✅ Agent の提案を信頼し、次の PR でも積極的に対応

### スコアが標準的な場合（⭐⭐⭐）
📌 コメント内容を人間が精査してから対応判定

### スコアが低い場合（⭐⭐ 以下）
⚠️ Agent のプロンプト・設定を改善検討

---
